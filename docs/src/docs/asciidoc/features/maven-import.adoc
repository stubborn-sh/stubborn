== Maven Import

The broker can import existing stubs JARs from Maven repositories (Nexus/Artifactory)
into its contract database. This enables teams to migrate from repository-based contract
sharing to broker-managed contracts without re-publishing every contract manually.

See specification: https://github.com/stubborn-sh/stubborn/blob/main/docs/specs/027-maven-import.md[docs/specs/027-maven-import.md]

=== One-Time Import

Import contracts from a specific stubs JAR by Maven coordinates:

[source,bash]
----
curl -X POST http://localhost:8642/api/v1/import/maven-jar \
  -H "Content-Type: application/json" \
  -u admin:admin \
  -d '{
    "applicationName": "order-service",
    "repositoryUrl": "https://nexus.example.com/repository/releases",
    "groupId": "com.example",
    "artifactId": "order-service",
    "version": "1.0.0"
  }'
----

The broker downloads the stubs JAR, extracts contracts and WireMock mappings from
`mappings/` and `contracts/` directories (including nested `META-INF/` paths), and
publishes each one via the existing contract publishing pipeline. This means imported
contracts trigger webhooks, appear in the matrix, and participate in can-i-deploy checks.

Re-importing the same version is safe (idempotent) — duplicate contracts are skipped.

=== Import Sources

Register a Maven repository + coordinates for tracking (future: automatic sync):

[source,bash]
----
# Register a sync source
curl -X POST http://localhost:8642/api/v1/import/sources \
  -H "Content-Type: application/json" \
  -u admin:admin \
  -d '{
    "repositoryUrl": "https://nexus.example.com/repository/releases",
    "groupId": "com.example",
    "artifactId": "order-service",
    "syncEnabled": false
  }'

# List registered sources
curl http://localhost:8642/api/v1/import/sources -u reader:reader

# Delete a source
curl -X DELETE http://localhost:8642/api/v1/import/sources/{id} -u admin:admin
----

=== REST API

[cols="1,2,2"]
|===
| Method | Path | Purpose

| POST | `/api/v1/import/maven-jar` | Import contracts from a stubs JAR
| GET | `/api/v1/import/sources` | List registered import sources (paginated)
| POST | `/api/v1/import/sources` | Register an import source
| GET | `/api/v1/import/sources/{id}` | Get source details
| DELETE | `/api/v1/import/sources/{id}` | Remove an import source
|===

=== Security

* URL scheme validation — only `http` and `https` are allowed (prevents SSRF via `file://`)
* JAR size limited to 50 MB, individual ZIP entries to 1 MB
* `ADMIN` or `PUBLISHER` role required for POST and DELETE operations
* `READER` role sufficient for GET operations
* Repository credentials stored with encryption (`encrypted_password` field)

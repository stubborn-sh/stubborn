= Branch Support

Contracts and verifications can be associated with a source code branch, enabling branch-aware workflows.

See specification: https://github.com/stubborn-sh/stubborn/blob/main/docs/specs/011-branches.md[docs/specs/011-branches.md]

== How It Works

When publishing a contract or recording a verification, an optional `branch` field tracks which
source code branch the artifact came from. This allows the broker to distinguish between
main-branch contracts and feature-branch contracts.

== Impact on Can I Deploy

Branch information is used in Can I Deploy checks and consumer version selectors to filter
which contracts are relevant for verification.

== Endpoints

The `branch` field is accepted on:

* `POST /api/v1/applications/{name}/versions/{version}/contracts` -- contract publishing
* `POST /api/v1/verifications` -- verification recording

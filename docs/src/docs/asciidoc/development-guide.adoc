= Development Guide

== Prerequisites

* JDK 25 (recommended: Liberica JDK via SDKMAN)
* Docker (for Testcontainers and dev mode Docker Compose)
* Maven 3.9+ (wrapper included: `./mvnw`)

== Build Commands

[source,bash]
----
# Format code (all Java modules)
./mvnw spring-javaformat:apply

# Full reactor build (all modules, all tests + static analysis)
./mvnw clean verify -T 1C

# Single module tests
./mvnw test -pl broker          # Broker unit tests
./mvnw test -pl stub-runner     # Stub Runner tests
./mvnw test -pl ui              # UI Vitest unit tests

# Integration tests (Docker required)
./mvnw failsafe:integration-test failsafe:verify -pl broker,e2e-tests

# Mutation testing
./mvnw org.pitest:pitest-maven:mutationCoverage -pl broker

# Run broker locally
./mvnw spring-boot:run -pl broker -Dspring-boot.run.profiles=dev

# UI development (hot reload)
cd ui && npm run dev

# Build Docker image
./mvnw spring-boot:build-image -pl broker -DskipTests
----

== Test Pyramid

|===
| Layer | Pattern | Runner | Docker?

| Unit tests | `*Test.java` | Surefire | No
| Property tests | `*Properties.java` | Surefire | No
| Controller slices | `*ControllerTest.java` | Surefire | No
| Repository slices | `*RepositoryTest.java` | Failsafe | Yes
| E2E tests | `*E2ETest.java` | Failsafe | Yes
|===

== Static Analysis

All 6 tools run during the build and fail on any violation:

1. License Header check — mycila license-maven-plugin (validate phase)
2. Spring Java Format — code formatting (validate phase)
3. Checkstyle — spring-javaformat-checkstyle rules (validate phase)
4. Error Prone + NullAway — compile-time bug detection (compile phase)
5. SpotBugs + FindSecBugs — bytecode analysis (verify phase)
6. PMD — complexity rules (verify phase)

== Architecture

Multi-module project with 15 Maven modules including `broker`, `broker-api-client`,
`ui`, `js`, `broker-stub-downloader`, `broker-contract-publisher`,
`broker-maven-plugin`, `broker-gradle-plugin`, `stub-runner`,
`stubborn-messaging-kafka`, `stubborn-messaging-rabbit`, and `e2e-tests`.

The broker module uses vertical slice architecture -- each feature is a self-contained package:

* `application` -- Application registration
* `contract` -- Contract publishing (with content hash deduplication)
* `verification` -- Verification results (with branch and pending support)
* `environment` -- Environment tracking
* `safety` -- Can I Deploy (with pending contract awareness)
* `graph` -- Dependency graph
* `webhook` -- Event webhooks (async delivery with retry)
* `tag` -- Version tagging
* `selector` -- Consumer version selectors
* `matrix` -- Compatibility matrix
* `maintenance` -- Data cleanup / retention
* `security` -- Cross-cutting security config (RBAC: READER/PUBLISHER/ADMIN)
* `observability` -- Metrics, health indicators, caching

The proxy module is an independent Spring Boot app for AI traffic-to-contract generation (Spring AI).

The UI module uses feature-sliced architecture (React 19 + Vite 6 + TailwindCSS 4 + TanStack React Query).
All list pages support server-side pagination, search, and default `createdAt,desc` sorting.
Application/version dropdowns use a searchable ComboBox with keyboard navigation.

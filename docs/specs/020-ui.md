# Cross-Cutting: Web UI

## What

A web dashboard embedded in the broker that provides a visual interface for all broker
features. The UI is served as static resources from the broker itself — no separate
web server needed.

## Why

Not all users interact with the broker through CI/CD pipelines or CLI tools. A web UI
enables:
- Teams to browse applications, contracts, and verifications without API calls
- Quick deployment safety checks via a form instead of curl commands
- Visual dependency graph exploration
- Webhook and tag management without memorizing API endpoints

## Pages

| Page | Purpose |
|------|---------|
| **Dashboard** | Overview with counts of applications, contracts, verifications, deployments |
| **Applications** | List, search, and register applications |
| **Contracts** | Browse contracts by application and version |
| **Verifications** | List and search verification results, record new verifications |
| **Environments** | View deployments per environment, record new deployments |
| **Can I Deploy** | Form to check deployment safety for an application version + environment |
| **Dependency Graph** | Visual graph of service dependencies derived from verifications |
| **Webhooks** | CRUD for event webhook subscriptions |
| **Compatibility Matrix** | Grid showing provider-consumer verification status |
| **Tags** | Add, remove, and browse version tags |
| **Data Cleanup** | Run cleanup with dry-run preview |
| **Settings** | Broker configuration and connection info |

## Shared Behaviors

### Pagination
All tables support server-side pagination with page navigation and configurable page size
(10, 20, 50). Default sort is newest first. Previous page data remains visible while the
next page loads (no loading flash).

### Search
Pages with many rows (applications, verifications, webhooks) have a search input that
filters server-side. Typing triggers a debounced API call. Search resets to page 0.

### Searchable Dropdowns
Application and version pickers are searchable combo boxes (not plain selects). Users can
type to filter, navigate with arrow keys, and select with Enter. This is important when
there are many applications or versions.

### Sorting
Tables backed by client-side data (matrix, graph) support column header click-to-sort.
Tables backed by server-side pagination do not support client-side sort because sorting
only the visible page would be misleading — the server controls sort order.

## Navigation

Single-page application with a sidebar navigation listing all pages. The UI is
feature-sliced: each page is self-contained with its own data fetching hooks.

## Authentication

The UI uses the same HTTP Basic or OAuth2 credentials as the API. If the broker requires
authentication, the browser's native auth dialog appears. The UI itself does not have a
custom login page.

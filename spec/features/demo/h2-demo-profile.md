# H2 Embedded Database for Demo Profile

## Problem

The demo deployment at demo.stubborn.sh uses Neon (hosted PostgreSQL) which:

1. **Costs money** for a free demo that only needs read-only access
2. **Adds an external dependency** that can go down independently
3. **Requires secrets management** (DATABASE_URL) in the deployment environment

## Solution

Switch the `demo` profile to use H2 in PostgreSQL compatibility mode. Since the demo is read-only (enforced by `DemoReadOnlyFilter`), we only need:

- Schema creation at startup
- Seed data loaded once
- Read endpoints working correctly

## Behavior

- When `SPRING_PROFILES_ACTIVE=demo`, the broker starts with an embedded H2 database
- H2 runs in PostgreSQL compatibility mode (`MODE=PostgreSQL`)
- Flyway loads H2-compatible migrations from `classpath:db/migration-h2`
- Demo seed data loaded from `classpath:db/demo-h2`
- JPA validation passes against the H2 schema
- All read-only API endpoints return correct data
- Write operations remain blocked by `DemoReadOnlyFilter`
- No external database dependency required

## What Changes

| Component | Before | After |
|-----------|--------|-------|
| Demo datasource | PostgreSQL (Neon) | H2 in-memory |
| Flyway migrations | `db/migration` (PG) | `db/migration-h2` (H2) |
| Demo seed data | `db/demo` (PG) | `db/demo-h2` (H2) |
| Deploy secrets | DATABASE_URL required | No DB secrets needed |
| Docker Compose | N/A (disabled) | N/A (disabled) |

## What Stays the Same

- Production and dev profiles still use PostgreSQL
- All existing tests using Testcontainers remain unchanged
- `DemoReadOnlyFilter` behavior unchanged
- API contract unchanged

## H2 Compatibility Notes

PostgreSQL features replaced for H2:

| PostgreSQL | H2 Replacement |
|------------|---------------|
| `gen_random_uuid()` | `RANDOM_UUID()` |
| `JSONB` | `CLOB` |
| `SERIAL` | `INT GENERATED BY DEFAULT AS IDENTITY` |
| `ON CONFLICT DO NOTHING` | Conditional INSERT |
| Partial indexes (`WHERE`) | Regular indexes |
| `setval()` / `nextval()` | H2 sequence syntax |
| `E'...'` escape strings | Standard strings |
